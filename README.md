# NeuroScript

NeuroScript это концепция учебного и сценарного языка программирования, в котором код читается почти как русский текст. Язык специально ограничен очень маленьким набором символов, чтобы синтаксис оставался естественным и легко проговаривался вслух.

## Идея языка

Цель NeuroScript дать русскоязычному новичку язык, где основная сложность переносится с символов на смысл.

- Вместо большинства операторов используются русские слова.
- Символьный синтаксис намеренно минимален и используется только там, где он резко упрощает запись.
- Каждая инструкция выглядит как короткое законченное предложение.
- Язык подходит для обучения, простых сценариев, визуального программирования и голосового ввода.

## Базовые правила

### Формат файла

Исходные файлы NeuroScript используют расширение `.ns`.

- `.ns` это основной формат программ NeuroScript
- файл хранится как обычный текст в кодировке `UTF восьм`
- один файл `.ns` содержит одну программу или один модуль
- интерпретатор читает файл сверху вниз
- пустые строки допустимы и служат только для читаемости

Примеры имен файлов:

```text
привет.ns
калькулятор.ns
цикл.ns
```

### Запуск файла

В проекте есть готовый интерпретатор `ns.py` и две обертки запуска:

- `ns.cmd` для `cmd`
- `ns.ps1` для `PowerShell`

Примеры запуска из корня проекта:

```text
ns.cmd examples\hello.ns
```

```text
.\ns.cmd .\examples\hello.ns
```

```text
.\ns.ps1 .\examples\hello.ns
```

Если в PowerShell запрещен запуск скриптов, можно использовать безопасный вариант:

```text
powershell -executionpolicy bypass -file .\ns.ps1 .\examples\hello.ns
```

### Разрешенные символы

В исходном коде NeuroScript разрешены только:

- буквы русского алфавита
- цифры
- пробел
- запятая
- точка
- знак `=`

Пример допустимой строки:

```text
имя = привет мир.
```

Пример недопустимой строки:

```text
имя := "привет"
```

### Окончание инструкций

Каждая инструкция обязательно заканчивается точкой.

```text
вывести привет.
счетчик = 1.
```

### Разделение аргументов

Части инструкции разделяются запятыми.

```text
имя = анна.
если возраст больше 18, вывести доступ открыт.
```

### Регистр и пробелы

- Язык регистронезависимый.
- Несколько пробелов подряд считаются одним разделителем.
- Переводы строк нужны только для удобства чтения.

### Идентификаторы

Чтобы синтаксис был однозначным, имена переменных и функций состоят из одного русского слова.

Допустимо:

```text
имя
счетчик
сумма
приветствие
```

Нежелательно и не рекомендуется:

```text
мое имя
первое число
```

## Типы данных

В первой версии языка есть два базовых типа.

### Число

Если значение состоит только из цифр, интерпретатор считает его числом.

```text
0
1
10
23
101
```

Примеры:

```text
возраст = 20.
счетчик = 1.
```

### Текст

Текст определяется автоматически. Если значение не похоже на число и не совпадает с именем существующей переменной, оно считается текстом.

```text
имя = анна.
приветствие = добрый день.
сообщение = привет мир.
```

Текстовые литералы в исходнике не содержат точку и запятую, потому что эти символы заняты синтаксисом языка.

### Автоматическое определение типа

NeuroScript определяет значение по форме записи, почти как Python.

- если значение состоит только из цифр, это число
- если значение является вызовом функции, используется возвращаемое функцией значение
- если значение это одно слово и такая переменная уже существует в текущей области видимости, берется значение переменной
- иначе значение трактуется как текст

Примеры:

```text
имя = анна.
возраст = 20.
копия = имя.
сообщение = добрый день.
```

## Переменные

### Присваивание

NeuroScript использует короткую форму, похожую на Python.

```text
имя = привет.
возраст = 20.
```

Правило простое:

- первое присваивание создает переменную
- повторное присваивание изменяет ее значение

```text
имя = здравствуй.
возраст = 21.
```

### Использование значения другой переменной

Имя переменной используется напрямую. Если имя уже существует в текущей области видимости, интерпретатор считает это ссылкой на переменную.

```text
копия = имя.
итог = сумма.
```

Если одно слово не объявлено как переменная, оно трактуется как текст:

```text
имя = анна.
город = москва.
```

## Выражения

NeuroScript поддерживает простые словесные выражения.

### Арифметика

```text
первое плюс второе
значение минус 1
```

Минимальный набор операций:

- `плюс`
- `минус`
- `умножить`
- `разделить`

Канонический пример:

```text
сумма = первое плюс второе.
```

### Сравнения

Для условий используются словесные сравнения:

- `равно`
- `больше`
- `меньше`

Примеры:

```text
возраст больше 18
ответ равно да
температура меньше 0
```

## Ввод и вывод

### Ввод

```text
ввести имя.
ввести возраст.
```

Правило интерпретации ввода:

- если введенное значение состоит из цифр, интерпретатор сохраняет число
- иначе значение сохраняется как текст

### Вывод

```text
вывести привет мир.
вывести имя.
вывести сумма.
```

## Условия

### Короткая форма

Одна инструкция после условия:

```text
если имя равно анна, вывести привет анна.
если возраст больше 18, вывести доступ открыт, иначе вывести доступ закрыт.
```

### Блочная форма

Для нескольких действий используется явное завершение блока.

```text
если ответ равно да.
вывести продолжаем.
иначе.
вывести остановка.
конец если.
```

## Циклы

Основной цикл строится вокруг слова `пока`.

```text
пока счетчик меньше 5.
вывести счетчик.
счетчик = счетчик плюс 1.
конец пока.
```

### Счетчики

Отдельный специальный синтаксис для счетчика не обязателен. Счетчик это обычная переменная, которую программа увеличивает внутри цикла.

```text
счетчик = 1.
пока счетчик меньше 6.
вывести счетчик.
счетчик = счетчик плюс 1.
конец пока.
```

## Функции

### Объявление функции

```text
функция сложить, параметры первое, второе.
вернуть первое плюс второе.
конец функции.
```

### Вызов функции

```text
результат = вызов функции сложить, аргументы 3, 4.
```

### Возврат значения

Инструкция `вернуть` завершает выполнение функции и отдает результат вызывающему коду.

## Примеры программ

### Привет мир

```text
вывести привет мир.
```

### Сложение двух чисел

```text
ввести первое.
ввести второе.
сумма = первое плюс второе.
вывести сумма.
```

### Цикл с выводом чисел

```text
счетчик = 1.
пока счетчик меньше 6.
вывести счетчик.
счетчик = счетчик плюс 1.
конец пока.
```

### Простая функция

```text
функция удвоить, параметры значение.
вернуть значение плюс значение.
конец функции.

результат = вызов функции удвоить, аргументы 5.
вывести результат.
```

Эти же примеры можно хранить в отдельных файлах с расширением `.ns`.

## Формальная модель синтаксиса

Ниже упрощенная грамматика в псевдо форме.

```text
программа                    := последовательность инструкций
последовательность инструкций := инструкция точка
                             | инструкция точка, последовательность инструкций

инструкция                   := присваивание
                             | ввод
                             | вывод
                             | если краткое
                             | если блок
                             | пока блок
                             | функция блок
                             | вернуть
                             | иначе метка
                             | конец метка

присваивание                 := имя = выражение
ввод                         := ввести имя
вывод                        := вывести выражение
если краткое                 := если условие, инструкция
если краткое                 := если условие, инструкция, иначе инструкция
если блок                    := если условие
блок если                    := инструкция точка
                             | инструкция точка, блок если
                             | иначе точка, блок если
                             | конец если
пока блок                    := пока условие
блок пока                    := инструкция точка
                             | инструкция точка, блок пока
                             | конец пока
функция блок                 := функция имя, параметры список параметров
блок функции                 := инструкция точка
                             | инструкция точка, блок функции
                             | конец функции
вернуть                      := вернуть выражение
иначе метка                  := иначе
конец метка                  := конец если
                             | конец пока
                             | конец функции

список параметров            := имя
                             | имя, список параметров

условие                      := выражение равно выражение
                             | выражение больше выражение
                             | выражение меньше выражение

выражение                    := неявное значение
                             | имя
                             | выражение плюс выражение
                             | выражение минус выражение
                             | выражение умножить выражение
                             | выражение разделить выражение
                             | вызов функции имя, аргументы список выражений

неявное значение             := число литерал
                             | текст литерал

число литерал                := последовательность цифр
текст литерал                := последовательность слов, не распознанная как число, имя переменной или вызов функции

список выражений             := выражение
                             | выражение, список выражений
```

Эта запись использует обычный метаязык для описания грамматики. В самой программе NeuroScript используются только реальные символы языка, перечисленные выше.
Для блочных конструкций открывающая инструкция тоже заканчивается точкой, а тело читается до служебных слов `иначе`, `конец если`, `конец пока` или `конец функции`.

## Правила интерпретации

### Выполнение программы

Интерпретатор выполняет инструкции сверху вниз.

### Области видимости

- переменные верхнего уровня живут в глобальной области
- параметры и локальные переменные функции живут внутри функции
- при завершении функции локальные переменные уничтожаются

### Присваивание и вычисления

- первое присваивание через `=` создает новую переменную в текущей области видимости
- повторное присваивание через `=` изменяет уже существующее значение
- голое значение из цифр интерпретируется как число
- одно голое слово сначала ищется как имя переменной, и только если переменной нет, трактуется как текст
- голая фраза из нескольких слов трактуется как текст, если не содержит служебную конструкцию языка
- арифметические операции работают только над числами
- если `плюс` применяется к текстам, его можно трактовать как склейку строк в расширенной версии языка

### Условия

- сравнение чисел происходит по числовому значению
- сравнение текста происходит по полному совпадению фразы
- истиной считается только явно истинное условие

### Вызов функции

- аргументы вычисляются слева направо
- значения параметров копируются в локальную область функции
- инструкция `вернуть` немедленно завершает функцию

## Как реализовать парсер

Для NeuroScript хорошо подходит простой рекурсивный нисходящий парсер.

### Шаг один, нормализация

- перевести текст в нижний регистр
- удалить лишние пробелы
- сохранить точки как разделители инструкций

### Шаг два, разбиение на инструкции

Программа делится по точкам. Пустые элементы отбрасываются.

Пример:

```text
имя = анна.
вывести имя.
```

Преобразуется в две инструкции.

### Шаг три, лексический разбор

Каждая инструкция разбивается на:

- слова
- числа
- запятые
- знак `=`

Так как набор символов очень маленький, полноценный сложный лексер не нужен.

### Шаг четыре, синтаксический разбор

Парсер смотрит на первое ключевое слово инструкции.

- если вторым токеном идет `=`, это разбор присваивания
- `ввести` значит разбор ввода
- `вывести` значит разбор вывода
- `если` значит разбор условной конструкции
- `пока` значит разбор цикла
- `функция` значит разбор объявления функции

Для блоков `если`, `пока` и `функция` парсер строит дерево до соответствующей закрывающей инструкции:

- `конец если`
- `конец пока`
- `конец функции`

Для голых значений парсер может использовать два режима:

- простой режим, где одиночное слово сразу считается ссылкой на переменную, если имя уже известно
- двухфазный режим, где парсер создает узел `НеявноеЗначение`, а окончательное решение принимает интерпретатор

### Шаг пять, построение дерева команд

Удобные типы узлов:

- `Программа`
- `Присваивание`
- `Ввод`
- `Вывод`
- `Если`
- `Пока`
- `Функция`
- `Возврат`
- `ДвоичноеВыражение`
- `СсылкаНаПеременную`
- `НеявноеЗначение`
- `ТекстовыйЛитерал`
- `ЧисловойЛитерал`
- `ВызовФункции`

### Шаг шесть, интерпретатор

После построения дерева команд интерпретатор обходит его и поддерживает:

- таблицу переменных
- стек вызовов функций
- механизм ввода и вывода

## Почему этот синтаксис реалистичен

NeuroScript остается похожим на обычный русский текст, но не пытается быть полностью свободным естественным языком. Язык специально вводит несколько якорных конструкций:

- `=`
- `вызов функции`
- `конец если`
- `конец пока`
- `конец функции`

Именно они делают синтаксис предсказуемым для парсера.

## Преимущества подхода

- очень низкий порог входа для русскоязычных новичков
- код читается как набор понятных команд
- присваивание стало коротким и привычным для тех, кто видел Python
- удобно объяснять базовые концепции программирования детям и начинающим
- можно строить голосовой или визуальный редактор поверх такого языка
- парсер получается проще, чем у большинства полноценных языков

## Ограничения подхода

- язык получается многословным
- без скобок и кавычек сложнее выражать вложенные конструкции
- текстовые литералы ограничены, потому что точка и запятая зарезервированы синтаксисом
- однословный текст может пересекаться с именем переменной, поэтому интерпретатору нужен приоритет разрешения имен
- идентификаторы лучше ограничивать одним словом, иначе резко растет неоднозначность
- математические выражения и сложные условия будут тяжелее читать, чем в традиционных языках
- язык ориентирован прежде всего на обучение и сценарии, а не на высоконагруженную разработку

## Возможное развитие

Во второй версии можно добавить:

- логические связки `и`, `или`, `не`
- списки и словари с полностью словесным синтаксисом
- встроенные конструкции `для каждого`
- модульную систему
- режим строгого и разговорного синтаксиса

## Итог

NeuroScript это язык, который намеренно жертвует компактностью ради ясности. Его сильная сторона не в скорости или лаконичности, а в том, что программа выглядит как осмысленный русский текст и может стать мостом между естественным языком и классическим программированием.
